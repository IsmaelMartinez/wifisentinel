Dashboard: Run Scan + Stream Results

The dashboard gets a "Run Scan" action. The Next.js API route spawns the CLI as a child process and streams scanner events back via Server-Sent Events. The dashboard renders results progressively — the topology map, host table, and score gauge all update as events arrive.

Results save to ~/.wifisentinel/scans/ automatically — same as running from the CLI

Implementation is straightforward: a new API route POST /api/scans/run spawns tsx src/cli.ts scan --events as a child process, piping the NDJSON event stream back as SSE. The dashboard subscribes with EventSource and updates its React state as events arrive. When the scan completes, the result is saved to the store and the page transitions to the normal scan detail view.
